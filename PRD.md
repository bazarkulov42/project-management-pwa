## 1. Обзор продукта

### 1.1 Проблема

ИТ-организация с двумя филиалами (Разработка и Обслуживание) испытывает системные проблемы:

- Документы, регламенты и должностные инструкции хранятся в разрозненных источниках без единой точки доступа
- Невозможно быстро проверить, соответствует ли задача или назначение структуре организации
- Загрузка сотрудников и сроки проектов не имеют единого визуального представления
- Потеря стратегических инициатив из-за отсутствия централизованного сбора идей
- Передача проектов между филиалами сопряжена с риском потери контекста

### 1.2 Решение

Мобильное приложение закрытого контура, объединяющее в одном интерфейсе:

- **Единое хранилище** документов, регламентов, проектов, оргструктуры и кадров
- **AI-движок** (Claude Sonnet 4.6) для проверки задач на соответствие базе знаний
- **Визуализацию нагрузки** — диаграмма Ганта + загрузка по сотрудникам
- **Закрытый контур** — пользовательские данные не покидают систему

### 1.3 Целевая аудитория

| Роль | Задачи в приложении | Ключевая ценность |
|---|---|---|
| Руководитель ЦУ | Мониторинг KPI, баланс нагрузки | Один экран вместо 5 отчётов |
| Руководитель филиала | Управление проектами и командой | Реальная загрузка команды |
| Менеджер проектов | Ведение реестра, передача проектов | Гант + дедлайны в мобильном |
| Аналитик / методолог | Проверка соответствия задач | AI-проверка за секунды |
| HR / кадровый отдел | Реестр людей, должностные инструкции | Статус ДИ по каждому сотруднику |

## 2. Цели и KPI

### 2.1 Бизнес-цели

1. Обеспечить единую точку доступа ко всей базе знаний организации
2. Сократить время проверки соответствия задачи с 30–60 минут до менее 30 секунд
3. Обеспечить равномерность загрузки между филиалами с разрывом не более 15%
4. Устранить риск потери стратегических инициатив через систему сбора идей
5. Гарантировать передачу проектов без потери контекста

### 3.2 Раздел «Документы» 📄

#### 3.2.1 Загрузка файлов

- Поддерживаемые форматы: **PDF, DOCX, DOC, XLSX, XLS, TXT**
- Множественная загрузка за один раз
- Автоматическое определение размера файла (КБ / МБ)
- Дата загрузки фиксируется автоматически
- Тег `«Загружено»` присваивается по умолчанию

#### 3.2.2 Добавление ссылок

- Поле **URL** (обязательное) — `https://` добавляется автоматически если отсутствует
- Поле **Название** (необязательное) — при пустом значении подставляется сам URL
- **Теги** через запятую — тег `«Ссылка»` добавляется автоматически
- Ссылки визуально отличаются от файлов: фиолетовая рамка, иконка 🔗, кликабельное название
- URL отображается в усечённом виде под названием (`max-width: 240px`, `text-overflow: ellipsis`)

#### 3.2.3 Управление

- Поиск по названию и тегам в реальном времени
- Удаление записи с уведомлением-toast
- Цветные теги с кастомными цветами по ключевым словам (ИБ, Политика, Основной, PMO, ITSM, Ссылка, Загружено)

### 3.7 Раздел «Нагрузка» 📊

Центральный инструмент балансировки ресурсов. Два режима — переключатель `Гант / Люди` в шапке.

#### Сводные метрики (верхние карточки, всегда видны)

| Карточка | Значение | Цвет |
|---|---|---|
| Средняя загрузка | % по всем сотрудникам | Зелёный / Жёлтый / Красный |
| Перегруженных | Кол-во сотрудников с загрузкой ≥ 110% | Красный если > 0 |
| Дедлайн ≤ 30д | Кол-во активных проектов | Жёлтый если > 0 |
| Всего проектов | Общий счётчик | Фиолетовый |

#### 3.7.1 Режим «Гант»

- Временная шкала: от самой ранней даты старта до самого позднего дедлайна
- Каждый проект — горизонтальная полоса с именем руководителя слева (12 символов + «…»)
- Палитра цветов: 7 уникальных цветов, назначаются по индексу
- **Просроченные проекты** (`deadline < сегодня` и `status ≠ Завершён`) — красный цвет полосы
- **Линия «Сегодня»** — красная пунктирная вертикаль с меткой `«сег.»`
- **Метки месяцев** по оси X с вертикальными серыми направляющими
- Чередующийся фон строк (#fafafa / #fff) для читаемости
- **Блок «Ближайшие дедлайны»** — список проектов, отсортированный по дедлайну:
  - 🟢 зелёная точка — дедлайн > 14 дней
  - 🟡 жёлтая точка — дедлайн ≤ 14 дней
  - 🔴 красная точка — просрочен

#### 3.7.2 Режим «Люди»

- Карточка на каждого сотрудника: аватар, ФИО, роль, % загрузки, прогресс-бар
- **Цветовая шкала загрузки:**
  - 🟢 Зелёный: 0–84%
  - 🟡 Жёлтый: 85–109%
  - 🔴 Красный: ≥ 110%
- Визуальное переполнение бара при > 100%
- **Автоматические предупреждения:**
  - `⚠ Перегрузка: N% сверх нормы` — если ≥ 110%
  - `⚡ Высокая нагрузка` — если 85–109%
- Список назначенных проектов под баром: `«Название проекта · Nч»`
- Если нет проектов — сообщение «Нет назначенных проектов»

#### 3.7.3 Формула расчёта загрузки

```
Загрузка (%) = ( Σ hours всех проектов, где lead === person.name ) / person.capacity × 100
```

- Ограничение для отображения: `Math.min(pct, 150)`
- Руководитель проекта (`lead`) сопоставляется с ФИО сотрудника точным совпадением строки

### 3.8 Раздел «Проверка» ✅

AI-движок на базе **Claude Sonnet 4.6**. Анализирует задачу на соответствие всей базе знаний организации.

#### 3.8.1 Входные данные

- Текстовое поле `textarea` для свободного ввода задачи, требования или назначения

#### 3.8.2 Контекст для AI (системный промпт)

В каждый запрос динамически передаётся актуальный срез хранилища:

```
ПРОЕКТЫ:     название (отдел, рук: ФИО, статус)
ДЕПАРТАМЕНТЫ: название (рук: ФИО, N чел.)
СОТРУДНИКИ:  ФИО — роль (отдел), ДИ: есть/нет
РЕГЛАМЕНТЫ:  название (статус)
ДОКУМЕНТЫ:   список названий
```

#### 3.8.3 Формат ответа AI (строгий JSON)

```json
{
  "verdict":         "соответствует" | "не соответствует" | "частично соответствует",
  "score":           0–100,
  "summary":         "строка (1–2 предложения)",
  "responsible":     "ФИО и роль (если определяется)",
  "issues":          ["массив строк"],
  "recommendations": ["массив строк"]
}
```

#### 3.8.4 Отображение результата

| Поле | UI-элемент |
|---|---|
| `verdict` | Цветной заголовок + иконка (✅ / ⚠️ / ❌) |
| `score` | Число `%` + прогресс-бар в цвет вердикта |
| `summary` | Текстовый абзац |
| `responsible` | Карточка «Ответственный» |
| `issues` | Красные плашки с буллетом `•` |
| `recommendations` | Голубые плашки со стрелкой `→` |

**Цвета вердикта:**

- `соответствует` → `#16a34a` (зелёный), фон `#f0fdf4`
- `частично соответствует` → `#d97706` (янтарный), фон `#fffbeb`
- `не соответствует` → `#dc2626` (красный), фон `#fef2f2`

- `max-height: 85vh` с вертикальной прокруткой для длинных форм
- Поля специфичны для каждого раздела
- Кнопка «Сохранить» в акцентном цвете раздела

### 4.4 Toast-уведомления

- Всплывающий баннер сверху по центру (`position: fixed, top: 16px`)
- Автоматически исчезает через **2800 мс**
- 🔵 Синий (`#1a6fd4`) — успешное добавление / загрузка
- ⚫ Тёмно-серый (`#374151`) — информация / удаление

### 4.5 Мобильная адаптация

- Максимальная ширина контейнера: **420px**, центрирован по горизонтали
- Нижний `padding: 80px` для навигационной панели
- Все интерактивные элементы: мин. **44px** по высоте (tap target)
- Карточки: `border-radius: 10–12px`, плоский дизайн без теней
- Шрифт: `'Segoe UI', system-ui, sans-serif` — без внешних загрузок

### 5.4 Расчёт загрузки (детально)

```javascript
// WorkloadBars / WorkloadSummary — useMemo
const loads = people.map(person => {
  const assigned  = projects.filter(p => p.lead === person.name);
  const totalHours = assigned.reduce((s, p) => s + (p.hours || 0), 0);
  const capacity   = person.capacity || 160;
  const pct        = Math.min(Math.round((totalHours / capacity) * 100), 150);
  return { ...person, totalHours, capacity, pct, projects: assigned };
});
```

## 6. Безопасность и закрытый контур

### 6.1 Принципы

- Все данные хранятся **исключительно в памяти устройства** (React state) — без серверов, без БД
- Никакие пользовательские данные не передаются во внешние системы, кроме AI-запросов
- AI-запросы содержат только **текстовые метаданные** (названия, роли) — без содержимого файлов
- Файлы анализируются только на клиенте (имя, размер через `File API`)
- Ссылки открываются в браузере устройства — приложение не проксирует трафик

### 6.2 Что передаётся в AI

| Передаётся ✅ | НЕ передаётся ❌ |
|---|---|
| Названия проектов | Содержимое файлов |
| Роли и ФИО сотрудников | Тексты регламентов |
| Статусы и даты | Персональные данные сверх ФИО/роли |
| Названия регламентов и документов | Финансовые данные |

### 6.3 Рекомендации по развёртыванию

- Деплой во **внутренней сети** организации (intranet)
- **HTTPS обязателен** для защиты AI-запросов в transit
- Аутентификация на уровне reverse proxy (Basic Auth / OAuth / SSO)
- API-ключ Anthropic — только в переменных окружения прокси-сервера
- Логирование AI-запросов — на усмотрение службы безопасности

## 7. Ключевые пользовательские сценарии

### Сценарий 1 — Проверка назначения

```
Актор:     Менеджер проектов / Аналитик
Цель:      Убедиться, что назначение соответствует структуре и компетенциям
Шаги:
  1. Открыть раздел «Проверка»
  2. Ввести текст задачи или назначения
  3. Нажать «Проверить соответствие»
  4. Получить вердикт + рекомендации
Результат: Вердикт с баллом, список проблем, имя ответственного — за 5–8 секунд
```

### Сценарий 2 — Контроль нагрузки

```
Актор:     Руководитель филиала
Цель:      Выявить перегруженных до того, как сроки сорваны
Шаги:
  1. Раздел «Нагрузка» → режим «Люди»
  2. Оценить прогресс-бары и предупреждения
  3. Переключиться в «Гант» для проверки сроков
  4. Принять решение о перераспределении
Результат: Визуальная картина загрузки за 10 секунд без ручного агрегирования
```

### Сценарий 3 — Добавление ссылки на регламент

```
Актор:     Методолог / Документовед
Цель:      Добавить ссылку на регламент во внутреннем Wiki в базу знаний
Шаги:
  1. Раздел «Документы» → кнопка «Добавить ссылку»
  2. Ввести URL и название
  3. Указать теги (например: «PMO, Wiki»)
  4. Сохранить
Результат: Ссылка доступна для поиска и используется AI при следующей проверке
```

### Сценарий 4 — Регистрация нового сотрудника без ДИ

```
Актор:     HR / кадровый специалист
Цель:      Зафиксировать нового сотрудника, отметить отсутствие ДИ
Шаги:
  1. Раздел «Сотрудники» → «+ Добавить»
  2. Заполнить ФИО, должность, отдел, норму часов
  3. НЕ отмечать чекбокс «Должностная инструкция загружена»
  4. Сохранить
Результат: Карточка с жёлтым бейджем ⚠ — видимый сигнал о незакрытом требовании
```

## 9. Ограничения и риски

| Риск | Описание | Митигация |
|---|---|---|
| Потеря данных при перезагрузке | State в памяти — данные не персистентны | v1.1: IndexedDB + экспорт JSON |
| Точность AI | Claude может давать неверный вердикт на редких сценариях | UI-дисклеймер «Рекомендуется ручная проверка» |
| Масштабирование данных | Большой context замедляет AI-ответ при 100+ записях | Пагинация контекста, сжатие метаданных |
| Зависимость от API | Проверка недоступна при отсутствии доступа к Anthropic | Graceful degradation, offline-режим в v1.2 |
| Однопользовательский режим | Нет разграничения доступа в v1.0 | Многопользовательский режим в v2.0 |
| Совпадение имён | `project.lead === person.name` — точное строковое сравнение | Ввод через `select` из списка людей в v1.1 |

## 10. Глоссарий

| Термин | Определение |
|---|---|
| **PRD** | Product Requirements Document — документ требований к продукту |
| **Закрытый контур** | Режим работы без передачи пользовательских данных на внешние серверы |
| **ЦУ** | Центральное управление — орган стратегического управления обоими филиалами |
| **ДИ** | Должностная инструкция сотрудника |
| **Гант** | Диаграмма Ганта — горизонтальная временная шкала проектов |
| **KPI** | Key Performance Indicator — ключевой показатель эффективности |
| **AI-вердикт** | Результат AI-проверки: соответствует / не соответствует / частично |
| **MTTR** | Mean Time To Restore — среднее время восстановления после инцидента |
| **SLA** | Service Level Agreement — соглашение об уровне обслуживания |
| **PWA** | Progressive Web App — веб-приложение с нативными возможностями |
| **Bottom sheet** | Мобильный UI-паттерн: модальное окно, выезжающее снизу экрана |
| **Норма часов** | Плановое количество рабочих часов сотрудника в месяц (по умолчанию 160) |
| **Toast** | Кратковременное всплывающее уведомление (исчезает через ~3 сек) |
| **Sticky header** | Заголовок, остающийся зафиксированным при прокрутке контента |
| **lead** | Поле проекта — ФИО руководителя; используется для сопоставления с реестром сотрудников |
